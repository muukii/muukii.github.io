Swift2.0コンパイラはarm64ビルド時のみ、さらに高速化されている

Swift1.2から2.0にアップし、コンパイル速度は40%ぐらいアップしていますが、
アーキテクチャごとのコンパイルでも違いが出ていました。
フルビルドのフローは同じ。

  1. Obj-C (CocoaPodsも含む)
  2. Swift
  3. Obj-C
  4. Link

(ちょっとだけ)Swiftファイルを変更した後の
ただ、インクリメンタルビルドの時は異なり、
armv7, armv7sをビルドの時

  1. Obj-C (キャッシュですぐ終わる) (~10秒ぐらい)
  2. Swift (差分のみコンパイル) (20~30秒ぐらい)
  3. Obj-C (Swiftと依存関係のあるファイル全て 今は270ファイルぐらい）(なんだかんだ20秒~)
  4. Link (10秒~)

と、インクリメンタルビルドとは言え、遅め
でもarm64だと

  1. Obj-C (キャッシュですぐ終わる)
  2. Swift (差分のみコンパイル)
  3. Link

最後のObj-Cコンパイルがないだけでかなり早いです。
これは本当に助かります。
あと、arm64だと他のアーキテクチャに比べてSwiftファイルのコンパイル自体も少し早いです。

今後さらなるコンパイルの高速化に期待しています。
